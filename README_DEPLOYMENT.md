### 📋 Deployment Steps Overview

1. **Prepare Locally** (5 min)
   - Update .env.production
   - Clear caches
   - Commit to Git

2. **Connect to Hostinger** (2 min)
   - SSH into server
   - Navigate to public_html

3. **Deploy Code** (10 min)
   - Clone repository
   - Install dependencies
   - Run migrations
   - Set permissions

4. **Configure Server** (5 min)
   - Upload .htaccess
   - Enable SSL
   - Configure domain

5. **Test** (5 min)
   - Verify site loads
   - Test functionality
   - Check admin panel

**Total Time:** ~30 minutes for first deployment

### 🔄 Updating Deployed Site

```bash
# On Hostinger server
cd domains/yourdomain.com/public_html
git pull origin main
composer install --no-dev --optimize-autoloader
php artisan migrate --force
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

**Update Time:** ~5 minutes

### 🆘 Troubleshooting

#### Common Issues

1. **500 Internal Server Error**
   ```bash
   tail -f storage/logs/laravel.log
   chmod -R 755 storage bootstrap/cache
   ```

2. **Database Connection Failed**
   - Verify .env database credentials
   - Check database exists in phpMyAdmin
   - Ensure user has proper permissions

3. **Images Not Loading**
   ```bash
   php artisan storage:link
   chmod -R 755 storage/app/public
   ```

4. **CSS/JS Not Loading**
   ```bash
   php artisan config:clear
   php artisan cache:clear
   php artisan config:cache
   ```

### 🔄 Rollback Procedure

If something goes wrong:

```bash
# Revert code
git log --oneline
git checkout [previous-commit-hash]

# Restore database
mysql -u user -p database < backup.sql

# Clear caches
php artisan config:clear
php artisan cache:clear
```

### 💾 Backup Strategy

#### Before Each Deployment
```bash
# Database backup
mysqldump -u user -p database > backup_$(date +%Y%m%d).sql

# Files backup
tar -czf backup_$(date +%Y%m%d).tar.gz storage public/storage
```
